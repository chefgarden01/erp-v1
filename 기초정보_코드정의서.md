## 1. 마스터상품 코드 (PRD)

>모든 상품의 기본 단위. SKU상품, 매핑, 시세조회 등 전체 시스템의 근간
>마스터상품을 관리하는 중량 단위는 상황에 따라 다름, 여러개의 중량단위를 선택해서 사용 및 분석 가능해야 함
>기본 중량단위는 KG이며 중량은 1000G임, 박스 4KG 중량일 경우 중량은 4000G임
>대분류,중분류,소분류 코드도 필요시 신규로 생성해서 사용
>아래 필드에서 추가 : 매입채널(농장,시장,외부)

## 2. SKU상품 코드 (SKU)

> 실제 판매되는 단위 상품. 마스터상품 1개에 여러 SKU 연결 가능
> SKU상품은 이미 판매처에 등록된 SKU코드를 엑셀로 대량으로 등록
> SKU상품 중 여러개의 마스터 혼합상품으로 구성된 경우가 있음
> 혼합상품에 구성되는 상품을 모두 매핑입력하고 해당상품별 출고중량 입력
> 출고중량은 정해진 중량만큼 실제 출고되지 않고 약간의 추가 투입량이 발생함
> SKU 상품별 입수중량, 추가중량, 출고중량으로 입력 필요
> SKU 상품에 증정상품이 있는 경우가 있음, 증정상품 선택시 추가 정보입력

| 필드명 | 설명 | 데이터타입 | 필수 | 비고 |
|--------|------|-----------|------|------|
| sku_id | SKU상품 코드 | VARCHAR(20) | ✅ | 자동생성 SKU-0001 |
| master_product_id | 마스터상품 코드 | VARCHAR(20) | ✅ | PRD 코드 참조 |
| sku_name | SKU상품명 | VARCHAR(150) | ✅ | 판매 상품명 |
| sku_weight | 소분중량 | DECIMAL | ✅ | g 또는 kg |
| weight_unit | 중량단위 | VARCHAR(5) | ✅ | g, kg |
| package_type | 포장형태 | VARCHAR(50) | ⬚ | 봉지, 박스, 팩 등 |
| selling_price | 판매가 | DECIMAL | ⬚ | 원 |
| discount_rate | 할인율 | DECIMAL | ⬚ | % |
| delivery_type_id | 배송타입 코드 | VARCHAR(20) | ⬚ | DLV 코드 참조 |
| shipping_method | 출고방식 | VARCHAR(20) | ⬚ | 직배송, 택배, 화물 |
| is_own_product | 직영상품 여부 | BOOLEAN | ✅ | 직영/위탁 구분 |
| order_deadline | 주문마감 시간 | TIME | ⬚ | 추가예상량 초기화 기준 |
| daily_optimal_qty | 1일 적정보유량 | DECIMAL | ⬚ | SKU 기준 보유량 |
| barcode | 바코드 | VARCHAR(50) | ⬚ | |
| thumbnail_url | 썸네일 이미지 | VARCHAR(255) | ⬚ | |
| detail_page_url | 상세페이지 URL | VARCHAR(255) | ⬚ | |
| registered_stores | 등록 스토어 목록 | TEXT | ⬚ | 쉼표 구분 |

| qr_code_url | QR코드 이미지 경로 | VARCHAR(255) | ✅ | 자동생성 |
| remarks | 비고 | TEXT | ⬚ | |

## 공통 규칙 정리

### 코드ID 자동생성 규칙
```
[접두어 3자리] + [-] + [일련번호 4자리]
예: PRD-0001, SKU-0023, CST-0105
```

### 공통 자동입력 필드 (전 코드 적용)
| 필드명 | 설명 | 입력방식 |
|--------|------|---------|
| created_by | 등록자 | 로그인 사용자 자동입력 |
| created_at | 등록일시 | 시스템 시간 자동입력 |
| updated_by | 수정자 | 수정 시 자동입력 |
| updated_at | 수정일시 | 수정 시 자동입력 |
| qr_code_url | QR코드 경로 | 코드 생성 시 자동생성 |

### QR코드 자동생성 규칙
- 코드 신규 등록 시 자동으로 QR코드 이미지 생성
- QR 스캔 → 해당 코드 상세화면으로 자동 이동
- QR북 출력 기능 지원

### 이력관리 규칙
- 모든 입력/수정 사항은 별도 이력 테이블에 자동 기록
- 이력 테이블: `{테이블명}_history` (변경전값, 변경후값, 변경사유, 변경자, 변경일시)

### 상태관리 규칙
- 삭제는 물리적 삭제가 아닌 상태값 변경 (소프트 삭제)
- 상태값: 사용 → 정지 → 삭제 (단방향 원칙, 정지→사용 복원 가능)

> 아래 항목을 확인하고 수정해 주세요:

2. **필드 추가/삭제**: 각 코드별 불필요한 필드 삭제, 빠진 필드 추가
3. **필수/선택 변경**: ✅/⬚ 표시 확인 후 변경
4. **데이터타입 변경**: VARCHAR 길이, 숫자타입 등 조정
5. **분류체계 수정**: 대분류/중분류/소분류 값 추가 또는 변경
6. **코드 통합/분리**: 유사 코드를 합치거나, 하나를 둘로 나누기
7. **새 코드 추가**: 목록에 없는 코드가 필요하면 추가
